## 🔍 Types of Metadata (Context)

### 1. **Code Metadata** (Gear Network - Not Used by Vara.eth)

**What it is:**
- Stored separately on the Gear network
- Contains: exports, static pages, instrumentation status
- Used by: Gear runtime for program management

**For Vara.eth:** ❌ Not needed - Vara.eth doesn't use this

### 2. **Sails IDL Metadata** (Optional)

**What it is:**
- Interface Definition Language file (`.idl`)
- Generated by Sails framework
- Used for: ABI generation, type-safe interfaces

**For Vara.eth:** ⚠️ Optional - Only needed if you want:
- Auto-generated Solidity interfaces
- Type-safe TypeScript bindings
- Better developer experience

**Your program:** Currently doesn't use Sails (not required)

### 3. **Runtime Metadata** (Substrate/Gear Runtime)

**What it is:**
- Metadata about the blockchain runtime itself
- Used by: Substrate/Polkadot tooling

**For Vara.eth:** ❌ Not relevant - Vara.eth runs on Ethereum

## 🔧 How Vara.eth Works Without Metadata

### Message Format

Your contract sends **raw bytes** to the Mirror:

```solidity
bytes memory payload = abi.encode(
    uint8(0),        // Action type
    marketId,        // Market ID
    initialYes,      // Initial YES amount
    initialNo,       // Initial NO amount
    block.number     // Block number
);

mirror.sendMessage(payload, false);
```

The WASM program **decodes** these bytes directly:

```rust
#[no_mangle]
extern "C" fn handle() {
    let input: Vec<u8> = msg::load_bytes().expect("Failed to load");
    
    // Decode manually based on your encoding scheme
    let action = input[0];
    match action {
        0 => initialize_market(&input[1..]),
        1 => execute_trade(&input[1..]),
        _ => panic!("Unknown action"),
    }
}
```

**No metadata needed** - the encoding/decoding is handled by your code!

## 📝 Optional: Adding Sails (If You Want Metadata)

If you want to generate ABIs and have better type safety:

### Step 1: Add Sails to Cargo.toml

```toml
[dependencies]
sails = "0.1.0"
# ... existing deps

[build-dependencies]
sails-build = "0.1.0"
```

### Step 2: Add Sails Annotations

```rust
use sails::sails;

#[sails::message]
pub enum Action {
    InitializeMarket { market_id: u64, initial_yes: u128, initial_no: u128 },
    ExecuteTrade { market_id: u64, user: [u8; 20], is_yes: bool, amount: u128 },
}

#[sails::message]
pub enum Reply {
    TradeResult { tokens_out: u128 },
    WithdrawalResult { eth_out: u128 },
}
```

### Step 3: Generate IDL

```bash
cargo build --release
# Generates: target/wasm32-gear/release/prediction_market_vara.idl
```

### Step 4: Generate Solidity Interface

```bash
cargo sails sol --idl-path target/.../prediction_market_vara.idl
```

**But this is OPTIONAL** - your current setup works fine without it!

## 💡 Why This Works

Vara.eth uses a **message-based protocol**:
- Your contract encodes messages as bytes
- WASM program decodes bytes directly
- No metadata/ABI needed - just agreed-upon encoding format

This is simpler than traditional approaches that require:
- ABI files
- Type registries
- Metadata schemas

**Your program works because:**
1. You control both sides (contract + WASM)
2. You define the encoding format
3. Both sides know how to encode/decode
